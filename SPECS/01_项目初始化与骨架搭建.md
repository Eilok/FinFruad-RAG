# 01_项目初始化与骨架搭建

## 阶段目标

建立满足约束的最小可运行项目骨架，确保前后端分离、目录职责清晰、依赖可管理、后续阶段可平滑扩展。

## 代码实现方法

1. 目录初始化
- 创建 `backend/` 与 `webui/`。
- 在 `backend/` 下预建：
  - `service/`（仅接口层）
  - `core/`（业务流程）
  - `storage/`（数据库与持久化）
  - `models/`（Pydantic 数据模型）
  - `providers/`（LLM 与 embedding 适配）
  - `scripts/`（离线任务与评估入口脚本）
- 在 `webui/` 初始化 Next.js 应用。

2. 后端最小服务
- 使用 FastAPI 提供健康检查接口：`GET /healthz`。
- 通过 `service/` 路由调用 `core/` 的占位逻辑，验证分层边界。

3. 配置系统
- 增加 `.env.example` 与 settings 模块。
- 配置项预留：LLM API、embedding 模型、Chroma 路径、日志目录、top-k 参数。

4. 依赖与运行
- 使用 `uv` 管理 Python 依赖。
- 完成后端启动脚本与前端启动脚本（README 记录）。

## 验收标准

- 存在 `backend/`、`backend/service/`、`webui/`，且目录职责与 AGENTS 约束一致。
- `backend/service/` 中不出现业务流程代码。
- 可成功启动 FastAPI 与 Next.js。
- `GET /healthz` 返回成功。
- README 包含本阶段安装与启动步骤（`uv` 命令）。
