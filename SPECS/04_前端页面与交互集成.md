# 04_前端页面与交互集成

## 阶段目标

构建可用的 Next.js 页面，支持文本输入、风险检测触发、结果展示与证据查看，打通前后端联调链路。

## 代码实现方法

1. 页面规划
- 首页提供文本输入框、检测按钮、参数区（如 top-k）。
- 结果区展示：是否诈骗、置信度、判别理由。
- 证据区展示：关键词召回与向量召回条目。

2. API 集成
- 封装前端 API 调用层，调用后端 `POST /detect`。
- 处理加载态、错误态、空结果态。

3. 交互体验
- 支持复制结果、清空输入、重试。
- 保留最近一次检测输入与结果（会话内）。

4. 工程规范
- 基础类型定义与响应 schema 同步。
- 前端环境变量配置后端地址。

## 验收标准

- 页面可提交文本并正确展示后端返回结果。
- 召回证据在前端可读且结构清晰。
- 异常时有明确提示，不出现白屏或崩溃。
- README 新增：前端启动方式、环境变量、联调步骤。
