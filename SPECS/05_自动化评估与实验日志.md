# 05_自动化评估与实验日志

## 阶段目标

实现基于 `job_scams` 与 `sms` 数据集的自动化评估流程，对比有/无 RAG 判别准确率，并输出完整实验日志。

## 代码实现方法

1. 数据读取与采样
- 解析 `data/job_scams` 与 `data/sms` 下 train/validation/test 的 jsonl。
- 从两个 train 中筛选 `label=1` 文本做离线构建语料。
- 从两个 test 中分别取前 500 条作为评估集。

2. 评估流程编排
- 步骤 A：执行离线构建（仅使用 train 的 label=1）。
- 步骤 B：无 RAG 评估（LLM 直判）。
- 步骤 C：有 RAG 评估（双检索 + 上下文增强）。

3. 指标计算
- 至少输出准确率。
- 建议补充 precision / recall / F1 / 混淆矩阵。
- 支持分数据集汇总与总体汇总。

4. 日志与产物
- 逐条日志字段：文本、真实标签、模型回答、预测标签、是否正确、模式（RAG/NoRAG）、时间戳。
- RAG 模式额外记录召回证据摘要与来源。
- 产出目录建议：`backend/outputs/eval/{run_id}/`。

5. 运行入口
- 在 `scripts/` 提供统一评估命令（可带参数：样本数、top-k、输出目录）。

## 验收标准

- 一条命令可完成完整对比实验（构建 + 双模式评估 + 指标汇总）。
- 两份数据集都被纳入测试，且各自 test 取前 500 条。
- 结果包含可追溯逐条日志与总体指标。
- README 新增：评估命令、日志说明、指标解读方式。
