# 02_离线知识库构建流水线

## 阶段目标

实现诈骗文本的结构化抽取、摘要向量化与 Chroma 入库，形成可持续增量构建的知识底座。

## 语言要求（新增）

- 文档与说明内容使用中文。
- 程序运行相关内容使用英文，至少包括：
  - LLM Prompt
  - API/CLI 报错信息
  - 日志中的状态值与提示文本
  - 对外可见的默认字段值（如 `category` 默认值）

## 代码实现方法

1. 数据模型定义
- 定义 `ScamAnalysis`：`summary`、`category`、`patterns`、`risk_keywords`。
- 定义 `KnowledgeRecord`：原文、结构化结果、时间戳、来源信息。

2. LLM 一次调用结构化抽取
- 在 `providers/` 编写 LLM 客户端。
- 设计严格输出提示词，要求返回 JSON。
- 增加 JSON 解析与容错（字段缺失兜底、重试机制）。
- 使用OpenAI SDK调用模型，留有base_url, api_key, model等参数配置接口。

3. 向量化与存储
- 在 `providers/` 增加 embedding 客户端。
- 在 `storage/` 封装 Chroma 操作：collection 初始化、写入、查询。
- 写入时保存：
  - document：`summary`
  - metadata：category、patterns、risk_keywords、original_text、source
- 使用硅基流动提供嵌入模型服务，调用接口规范如下：
```bash
curl -X POST https://api.siliconflow.cn/v1/embeddings \
  -H "Authorization: Bearer $SILICONFLOW_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "input": "Hello, world!",
    "model": "Qwen/Qwen3-VL-Embedding-8B"
  }'
```

```json
{
  "object": [
    "list"
  ],
  "model": "<string>",
  "data": [
    {
      "object": "embedding",
      "embedding": [
        123
      ],
      "index": 123
    }
  ],
  "usage": {
    "prompt_tokens": 123,
    "completion_tokens": 123,
    "total_tokens": 123
  }
}
```

4. 离线构建入口
- 在 `scripts/` 提供构建脚本：支持单条文本、批量文本、增量导入。
- 可通过 CLI 参数控制批次大小、重试次数、输出日志位置。

5. 服务接口
- 在 `backend/service/` 暴露离线构建 API（例如 `POST /kb/ingest`）。
- 接口层只做参数校验与调用编排。

## 验收标准

- 给定任意诈骗文本，可得到结构化结果且字段完整。
- `summary` 已生成 embedding 并成功写入 Chroma。
- Chroma 中可查询到对应 metadata（含关键词、套路、原文）。
- 离线脚本可批量运行并输出构建日志。
- README 新增：离线构建命令、输入格式、输出位置、常见报错排查。
