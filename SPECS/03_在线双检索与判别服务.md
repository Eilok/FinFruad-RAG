# 03_在线双检索与判别服务

## 阶段目标

实现关键词检索 + 向量检索双通道召回，融合证据后辅助 LLM 对用户文本进行诈骗风险判断。

## 代码实现方法

1. 在线输入处理
- 定义输入模型：用户文本、top-k 参数、是否返回证据。
- 预处理：分词/关键词提取、文本规范化。

2. 关键词检索通道
- 从输入文本提取关键词。
- 在知识库记录的 `risk_keywords` 中进行匹配打分。
- 返回关键词检索 top-k。

3. 向量检索通道
- 对用户文本生成 embedding。
- 在 Chroma 对 `summary` 向量做相似度查询。
- 返回向量检索 top-k（含相似度分数）。

4. 结果融合
- 合并两路召回并去重。
- 统一打分（可先采用加权规则，后续可配置）。
- 生成给 LLM 的证据上下文（摘要、类别、套路、关键词命中说明）。

5. LLM 最终判别
- 设计输出 schema：`is_scam`、`confidence`、`reason`、`evidence_refs`。
- 在 `service/` 暴露在线接口（例如 `POST /detect`）。

## 验收标准

- 输入任意文本时可返回两路检索结果与融合结果。
- 判别接口可输出结构化结论（含是否诈骗、理由、置信度）。
- 接口支持配置 top-k 并稳定返回。
- 失败场景（空文本、模型超时、无召回）有可读错误处理。
- README 新增：在线接口调用示例与返回字段解释。
