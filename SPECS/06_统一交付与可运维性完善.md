# 06_统一交付与可运维性完善

## 阶段目标

在功能完整后进行统一收口，确保项目具备一致的使用入口、清晰文档与可维护运行体验。

## 代码实现方法

1. 统一入口
- 后端、前端、离线构建、在线检测、自动评估分别提供统一命令入口。
- 可选增加 `Makefile` 或脚本聚合命令（Windows 可提供 `.ps1`）。

2. 文档收口
- README 重构为：
  - 项目简介
  - 架构总览
  - 快速开始
  - 配置说明
  - 离线构建
  - 在线检测
  - 自动评估
  - 常见问题
- 明确每个输出目录用途。

3. 健壮性与可观测
- 增加关键日志、异常分类、重试与超时配置。
- 对核心接口补充基础测试样例。

4. 版本与发布准备
- 检查依赖一致性（`uv`）。
- 清点必要环境变量与默认值。
- 形成最小发布检查清单。

## 验收标准

- 新用户按 README 可在干净环境完成端到端运行。
- 所有阶段命令在文档中有统一入口和示例。
- 评估输出、日志输出、向量库路径均可定位。
- 项目结构仍严格满足 `backend/service` 只放接口层代码的约束。
